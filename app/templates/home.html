Welcome to Our Japanese Food Website

  1. The database also has a one-to-many relationship (sauce) - this might be enough for Y12s
  2. There are descriptions for toppings - this could be extended (GF, DF, photos etc)
  3. The entire Flask app code is in one file (main.py) - config (at the very least) belongs in its own file - See Teddy Bears Picnic for some ideas around that
  4. There are no photos in this version - again, see TBP for ideas around that - photos can be stored as BLOBs in the database - but probably shouldn't be in this instance - use the STATIC folder
  5. The static folder is the only part of any Flask app that users (i.e., the browser) can access, everything else is secured and only available to the Flask app - so CSS, JS, images, logos, etc., all need to be in there.
  6. I have used inline CSS - this is generally a very bad idea - separation of concerns... content should be in HTML, style in CSS. This is not a web design course, it's all about the Flask.
  7.
